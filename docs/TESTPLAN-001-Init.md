# TESTPLAN-001 v0

## Цели
- Проверить критические пользовательские сценарии и базовую идемпотентность API.

## Сценарии (позитив)
- Login -> token
- License OK (есть квота)
- Reserve -> Commit
- Reserve -> Rollback
- Free plan 50/мес

## Сценарии (негатив)
- Токен невалиден (401)
- Квота 0 (402 Payment Required)
- Двойной commit (409)
- Просроченная reservation (409/422)
- Rate limit 429
- Desktop офлайн: поведение при недоступном /license

## Нагрузка
- 50 параллельных резервов/пользователя, проверка идемпотентности по correlation_id.

## Критерии приёмки
- p95 API < 300ms
- 0 критических багов
